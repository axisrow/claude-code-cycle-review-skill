# CLAUDE.md

## Project Overview

Claude Code plugin providing the `/cycle-review` skill ŌĆö an automated PR review loop with **two modes**. On first run it onboards the user to pick which review bots they have (`@claude`, `@codex`, or both) and a default review **mode** (`cloud`/`local`). Plans a multi-PR merge strategy when several PRs are open, verifies each PR implements its linked issue 100% before requesting review (step 3), gets a review, triages comments/findings (FIX / SKIP / HALLUCINATION / IRRELEVANT / CONFLICTING / ALREADY_FIXED / UNVERIFIED), applies fixes, and repeats until approval.

- **Cloud mode** (default, original behavior): pings the configured GitHub bots in a PR comment, waits for them via `wait-for-reviews.sh`, then runs autonomously through CI + squash-merge.
- **Local mode**: the built-in `/review` command (invoked via the **`Skill` tool**, read-only single-pass; its review lands as an assistant message in the skill's own context and is read from there by judgment ŌĆö NOT a PR comment, no fixed format) reviews the PR ŌĆö and when `@codex` is in the reviewers list it *also* runs Codex locally via the codex plugin's `codex-companion.mjs adversarial-review --base <PR baseRefName> --json` (run in the background, in parallel; findings merged before triage; **fail-closed** if Codex is configured but unavailable, or if `/review` returns no review at all). No bot ping, no GitHub wait. Each round's merged findings (both `/review` and Codex) are persisted to `~/.claude/cycle-review/runs/<PR>-round<N>.json` so the step-9 cleanup pass can re-read across rounds. It records each round's verdicts as a PR summary comment, fixes, commits, and pushes, but is **review-only on merge**: it never merges on its own; the user triggers merge explicitly.

## Key Details

- A single skill, `/cycle-review`, invoked directly (no slash-command alias)
- Requires `gh` CLI authenticated with GitHub, plus `jq` for reading/writing the onboarding config
- **Two review modes (step 1):** `cloud` (default) and `local`. Selection precedence: a leading `local`/`cloud` (or `--local`/`--cloud`) token in `$ARGUMENTS` overrides the saved `mode`; with neither, default to `cloud` (backward-compatible, incl. v1 configs without a `mode` field). The token is stripped before PR-number parsing, same as `onboard`. Local mode reviews with the built-in `/review` command, plus Codex locally when `@codex` is configured (parallel, merged, fail-closed), posts a triage-summary comment per round, and **never auto-merges** (review-only on merge ŌĆö user triggers it). Cloud mode is the original autonomous-through-merge path
- Onboarding (step 0): optional, asks an `AskUserQuestion` for reviewers (multi-select `@claude`/`@codex`, used by cloud) + default mode (single-select `cloud`/`local`). Stored globally at `~/.claude/cycle-review/config.json` (`{"reviewers": [...], "mode": "cloud", "version": 3}`). v1/v2 configs stay valid (mode absent ŌćÆ cloud). Re-run with `/cycle-review onboard` (legacy: `--onboard` / `--reconfigure`). Cloud steps 4ŌĆō5 are driven by the reviewers list ŌĆö never hardcode `@claude`. **Dev mode:** if the installed codex plugin is the fork (the companion path contains a `-fork/` segment), a second `AskUserQuestion` asks for default Codex model (`spark`/`sol`/`terra`/`luna`) + effort (`medium`/`high`/`xhigh`/`max` ŌĆö picker capped at 4 per Claude Code's limit; the rare `none`/`minimal`/`low`/`ultra` stay reachable via `--effort` per-run override), stored as `codex_model`/`codex_effort` and passed to the companion every round. Upstream `codex` ŌĆö no dev mode, unchanged.
- Both configured ŌåÆ one ping comment starting `@claude @codex`; single reviewer ŌåÆ just that mention
- **Step 5 waiting is a deliberately dumb driver** `skills/cycle-review/wait-for-reviews.sh` (~50 lines). It just `sleep`s a fixed window (`WAIT`, default 300s) then probes that the API is reachable, and prints one line: `DONE` (wait elapsed, API up ŌåÆ triage) or `ERROR` (sustained API outage ŌåÆ stop, don't merge). It does NOT detect who finished, parse bodies, track per-reviewer state, or resume ŌĆö ALL interpretation (who replied, relevance, hallucination, Claude usage-limit) happens in the step-6 triage, which reads every comment from every surface itself. No state file, no `RESET`. Run via Bash with `run_in_background: true` + `dangerouslyDisableSandbox: true` (leading foreground `sleep` is blocked; `sleep` inside the backgrounded script is fine). This replaced an over-engineered ~290-line version whose completion-detection/round-scoping/resume machinery kept generating its own bugs
- Claude usage-limit is handled in triage, not a separate step: a usage-limit message means "Claude did not review" (not a finding); proceed on Codex if it reviewed, else stop. Never wait for the limit to reset
- Safety invariant lives in step-6 finalize: never merge unless at least one reviewer actually reviewed this round; `ERROR` from step 5 or bot silence ŌćÆ stop (an empty triage is not approval)
- Completeness gate (step 3): before requesting review, verify each PR implements its linked issue's design 100% (read the issue, checklist the design, verify the diff, close gaps test-first). Bots check code correctness, not completeness against the issue ŌĆö don't burn a review round on a half-finished PR
- Final cleanup pass (step 9): the round with no `FIX` verdicts is the last cycle. Before CI+merge, apply ALL accumulated minor findings ŌĆö every genuine `SKIP` plus reasonable nice-to-haves ŌĆö gathered from EVERY prior round (not just the last), de-duped; exclude `HALLUCINATION`/`IRRELEVANT`/`CONFLICTING`/`ALREADY_FIXED`/`UNVERIFIED`. Lint+test+commit, then go straight to merge ŌĆö no extra review round for the cleanup
- Max 3 cycles per PR (one cycle = one steps 4ŌĆō8 round). If a 3rd cycle still has `FIX` verdicts, stop instead of looping a 4th time: summarize the open findings and ask the user to narrow scope (move some out of scope into a follow-up issue/PR, or rethink). Cap only bites when findings persist; a clean 1st/2nd round finalizes normally
- All `gh` commands need `dangerouslyDisableSandbox: true` (sandbox blocks TLS to api.github.com)
- Skill uses Agent tool for comment triage (subagent)
- **Local Codex (step 4):** when `@codex` is in the reviewers list, local mode also runs Codex. Companion path is resolved from `~/.claude/plugins/installed_plugins.json` (the canonical pluginŌåÆinstallPath index). Filter hard ŌĆö the chosen script runs with `dangerouslyDisableSandbox: true`, so a wrong pick can execute an unrelated plugin: only entries whose plugin name is exactly `codex` or `codex-fork` (anchored allowlist, not substring), only `scope: "user"` (never `local`/`project` scoped to another repo), newest `lastUpdated` first (actively installed version, not a file-cache leftover), then probe both layouts (`<installPath>/plugins/codex/scripts/codex-companion.mjs` for the fork meta-plugin, `<installPath>/scripts/codex-companion.mjs` for the direct upstream) and take the first existing. Never hardcode the source namespace or path layout. (`${CLAUDE_PLUGIN_ROOT}` can't be used here ŌĆö it points at the owning plugin, cycle-review, not codex.) It reviews the PR diff with `adversarial-review --wait --json --base <baseRefName>` (`baseRefName` from `gh pr view`; works because local mode is checked out on the PR branch ŌĆö companion can't review a PR by number, only local git). **Dev mode (codex-fork):** the companion path containing a `-fork/` segment unlocks `--model`/`--effort` flags, sourced from config (`codex_model`/`codex_effort`) with per-run `--model`/`--effort` override; upstream `codex` gets neither flag. Run in the background (`Bash run_in_background: true` + `dangerouslyDisableSandbox: true`) in parallel with `/review`. Output parsed from **`.result.findings`** (not `.findings`), mapped to the common finding shape (`file`ŌåÆ`path`, `line_start`ŌåÆ`line`, severity critical/highŌåÆcritical else minor), tagged `reviewer: codex`, merged with `/review`'s findings (read by judgment from the free-form prose `/review` writes as an assistant message in the skill's own context ŌĆö `/review` posts nothing to GitHub; invoked via the Skill tool; no fixed format, no steering, no parser), then triaged (Codex claims re-verified, not trusted). Fail-closed: empty path (codex plugin not installed ŌĆö say so, don't suggest `codex login`), non-zero exit (installed but not logged in ŌĆö ask for `codex login`), OR the `Skill` `/review` call erroring / producing no review at all ŌćÆ stop, never fall back to `/review` alone, never finalize on silence
- Language: skill prompts in English, responds in user's language
